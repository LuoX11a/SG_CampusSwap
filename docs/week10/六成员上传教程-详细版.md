# SG CampusSwap — 六成员上传教程（零基础版）

> 不需要任何 GitHub 使用经验。跟着每一步操作即可。
> 全程约 30 分钟。严格按顺序，一个人做完下一个人再做。

---

# 第 0 步：所有人先做 — 下载项目到电脑

## 你要做什么
把 GitHub 上的项目下载到你的电脑桌面。

## 操作步骤

### 如果你用的是 Windows

**1. 打开文件资源管理器**
按键盘上的 `Win + E` 键（Win 键是键盘左下角那个 Windows 图标键）

**2. 进入桌面**
在左侧点击「桌面」

**3. 在桌面空白处右键 → 打开 Git Bash Here**
如果你右键菜单里没有这个选项，说明没装 Git，请看下方「没装 Git 怎么办」

**4. 在弹出的黑色窗口里输入下面这行命令：**
```bash
git clone https://github.com/LuoX11a/SG_CampusSwap.git
```
然后按回车键。

等几十秒，桌面上会出现一个叫 `SG_CampusSwap` 的文件夹。这就是项目。

**5. 进入项目文件夹**
```bash
cd SG_CampusSwap
```

**6. 切换到 develop 分支**
```bash
git checkout develop
```

**7. 拉取最新代码**
```bash
git pull origin develop
```

### 如果你用的是 Mac

**1. 打开访达（Finder）→ 桌面**

**2. 按 `Command + 空格` 打开搜索，输入 `终端`，打开终端**

**3. 在终端输入：**
```bash
cd ~/Desktop
git clone https://github.com/LuoX11a/SG_CampusSwap.git
cd SG_CampusSwap
git checkout develop
git pull origin develop
```

### 没装 Git 怎么办
- Windows: 去 https://git-scm.com/download/win 下载安装，一路点 Next
- Mac: 打开终端输入 `xcode-select --install`，然后点安装

# 第 1 步：Wang Bowei（后端负责人）

> ⏱ 大约 5 分钟 | 必须第一个做

## 你要上传的东西
- 8 个后端 API 路由文件
- 1 个主程序文件
- 3 个 Docker 配置文件

## 操作步骤

### 1. 确认项目是最新的
打开 Git Bash（或 Mac 终端），输入：
```bash
cd ~/Desktop/SG_CampusSwap
git checkout develop
git pull origin develop
```
看到 `Already up to date.` 说明已经是最新的。

### 2. 打开 source 文件夹（我们要复制的东西在这里）
打开文件资源管理器，进入：
```
桌面 → week10 → Wang-Bowei → backend → app → api → v1
```
这个文件夹里面有 8 个 `.py` 文件：
`__init__.py`, `auth.py`, `chat.py`, `items.py`, `reviews.py`, `search.py`, `upload.py`, `users.py`

选中这 8 个文件，按 `Ctrl+C` 复制。

### 3. 打开目标文件夹（要粘贴到哪里）
打开另一个文件资源管理器窗口，进入：
```
桌面 → SG_CampusSwap → backend → app → api
```
如果里面没有 `v1` 文件夹：右键 → 新建文件夹 → 命名为 `v1`

进入 `v1` 文件夹，按 `Ctrl+V` 粘贴。

### 4. 覆盖 main.py
回到 source 文件夹：
```
桌面 → week10 → Wang-Bowei → backend → app
```
复制 `main.py`

到目标文件夹：
```
桌面 → SG_CampusSwap → backend → app
```
粘贴，选择「替换目标中的文件」。

### 5. 复制 Docker 文件
复制 source 文件夹里的：
```
桌面 → week10 → Wang-Bowei → Dockerfile
桌面 → week10 → Wang-Bowei → docker-compose.yml
桌面 → week10 → Wang-Bowei → .env.example
```

分别粘贴到：
```
Dockerfile       → 桌面 → SG_CampusSwap → backend → Dockerfile
docker-compose.yml → 桌面 → SG_CampusSwap → docker-compose.yml
.env.example     → 桌面 → SG_CampusSwap → backend → .env.example
```
.env.example 粘贴时选择「替换目标中的文件」。

### 6. 上传到 GitHub
打开 Git Bash（或 Mac 终端），逐条输入以下命令（每输完一条按回车）：

```bash
cd ~/Desktop/SG_CampusSwap
```

```bash
git add backend/app/api/v1/ backend/app/main.py backend/Dockerfile docker-compose.yml backend/.env.example
```

```bash
git commit -m "feat: add API routes, Docker config, and health check"
```

```bash
git push origin develop
```

看到类似 `Writing objects: 100%` 就说明上传成功了。

### 7. 群里通知
在微信群里发：**「Wang Bowei 上传完成，Wang Xu 可以开始了」**

# 常见问题

## Q: 我输入 git push 的时候弹出登录窗口
A: 输入你的 GitHub 用户名和密码。如果密码不对，需要去 GitHub 网站 → Settings → Developer settings → Personal access tokens → 生成一个 token 当作密码用。

## Q: 显示 "error: failed to push"
A: 说明在你上传期间有人先上传了。重新执行：
```bash
git pull origin develop
git push origin develop
```

## Q: 显示 "merge conflict"
A: 在群里说一声，让 Huang Hongrui 来处理。

## Q: 我不小心关掉了 Git Bash 窗口
A: 没关系，重新打开一个，输入 `cd ~/Desktop/SG_CampusSwap` 回到项目目录继续操作。

## Q: 源文件夹（week10）在哪里
A: 在桌面的 `cp3102 → week10` 里面。如果你找不到，在文件资源管理器搜索 `week10`。

## Q: 我怎么知道上传成功了
A: 打开 https://github.com/LuoX11a/SG_CampusSwap ，看看你的文件是不是出现在里面了。
